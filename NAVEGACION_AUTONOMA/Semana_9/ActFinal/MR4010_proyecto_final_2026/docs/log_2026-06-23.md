## 2. Entrenamiento del modelo CIL

### Arquitectura (Codevilla 2018)

- **Entradas**: imagen 200×88 RGB, velocidad normalizada (÷30), comando one-hot float `(1,4)`
- **Backbone**: 8 conv (ELU) → 2 FC (512) → 4 ramas de 256+1 neuronas (CONTINUE/LEFT/RIGHT/STRAIGHT)
- **Selección de rama**: `Multiply([all_branches, cmd_one_hot])` → `Dense(1, kernel_initializer='ones', trainable=False)`

### Fixes críticos al notebook `train_cil.ipynb`

| Bug                              | Fix aplicado                                                  |
|----------------------------------|---------------------------------------------------------------|
| `Lambda(tf.one_hot)` — Keras 3 no puede inferir shape | `cmd_in` acepta one-hot float pre-computado, sin Lambda |
| `Lambda(reduce_sum)` — no serializable en `.h5` | Reemplazado por `Dense(1, trainable=False)`            |
| `BASE` path detection fallaba    | Hardcoded a ruta absoluta del proyecto                        |
| `load_model` safe_mode error     | Añadido `safe_mode=False`                                     |
| `np.array([nav_cmd], dtype=int32)` en inferencia | Cambiado a one-hot float `(1,4)`               |

## 4. Bugs identificados — pendientes de fix

### Bug 1 — Gyro usa índice [2] (incorrecto para BMW)
**Ubicación**: `autonomous_cil.py:312`
```python
# ACTUAL (incorrecto):
heading_accum += gyro_vals[2] * timestep / 1000.0

# CORRECTO (validado en Act 4.2):
heading_accum += gyro_vals[1] * timestep / 1000.0
```
El BMW reporta yaw en el eje Y (índice `[1]`), no Z (`[2]`). Con índice 2 siempre es cero → el REORIENT nunca corrige el heading → el carro se queda girando indefinidamente.

---

### Bug 2 — Condición de salida de EVADE_OBS está invertida
**Ubicación**: `autonomous_cil.py:386`
```python
# ACTUAL (lógica incorrecta):
if ds_r_val > DS_CLEAR_DIST and ds_f_val < DS_ENGAGE_DIST:

# CORRECTO — salir cuando el bus está atrás Y el frente derecho está despejado:
if ds_r_val > DS_CLEAR_DIST and ds_f_val > DS_ENGAGE_DIST:
```
Con el operador `<`, la condición requiere que el sensor frontal-derecho VEA algo (<4.5m) mientras el trasero está libre — estado imposible cuando el bus ya pasó. El carro se queda atrapado en EVADE_OBS indefinidamente y sigue girando hasta chocar.

---

### Bug 3 — No hay doble confirmación Recognition + LiDAR para activar evasión
**Ubicación**: `autonomous_cil.py:337`
```python
# ACTUAL (solo LiDAR, falsos positivos posibles):
elif lidar_d < OBS_LIDAR_THRESH:
    state = STATE_EVADE

# MEJOR (igual que Act 4.2): agregar confirmación de Recognition
# Verificar que hay un objeto tipo Bus antes de evadir
```
Sin confirmation por Recognition, postes y edificios en el LiDAR pueden activar el EVADE innecesariamente. En Act 4.2 se usó `BUS_CONFIRM_FRAMES=1` con color exacto.
